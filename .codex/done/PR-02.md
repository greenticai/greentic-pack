## Goal
Delete the legacy `greentic-pack events ...` command group. Add `greentic-pack providers ...` that operates on the provider extension stored in `PackManifest.extensions` using greentic-types helpers.

## Dependencies
- Requires greentic-types exposing:
  - `PROVIDER_EXTENSION_ID`
  - `PackManifest::provider_extension_inline()`
  - `ProviderExtensionInline::validate_basic()`

## CLI
Add:
- `greentic-pack providers list [--pack <path>] [--json]`
- `greentic-pack providers info <provider_id> [--pack <path>] [--json]`
- `greentic-pack providers validate [--pack <path>] [--strict]`

Remove:
- `greentic-pack events` (all subcommands + docs + tests)

## Behavior
### Pack resolution
- If `--pack` omitted: use current directory.
- Locate pack manifest using the existing greentic-pack logic (whatever file name it already supports).
- Parse into `greentic_types::PackManifest`.

### Provider extension resolution
- Use `manifest.provider_extension_inline()` to fetch provider list.
- If no provider extension exists:
  - `providers list` prints empty list and exits 0
  - `providers info` errors with "provider extension not present"
  - `providers validate` succeeds unless `--strict` and a policy says it must exist (default: succeed)

### Output
- Default human output:
  - `list`: stable order by provider id, columns: ID, RUNTIME, KIND (if available), DETAILS (short)
  - `info`: pretty-print the ProviderDecl + resolved runtime ref (serde_yaml or pretty JSON)
- `--json` output:
  - list => JSON array of ProviderDecl objects (or a normalized object with selected fields)
  - info => JSON object

### Validation
- `providers validate`:
  - calls `ProviderExtensionInline::validate_basic()`
  - adds CLI-only checks:
    - if ProviderDecl references local files/paths (if such fields exist), ensure they exist relative to pack root
  - `--strict`: warnings become errors (if validate_basic differentiates; otherwise treat any issue as error)
